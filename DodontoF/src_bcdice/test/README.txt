このディレクトリのデータはテスト用のデータになります。

主だったファイルは以下の通り

・test.rb
B&C2.x系（本媒体）のテスト用スクリプト

ruby -Ku test.rb

のように実行することで、BCDiceのユニットテストを実行することができます。
改造後はテストを追加して必ず実行しましょう！
テスト用データについては次項の testData.txt を参照。

・testData.txt
test.rb で実行するテストデータ
例えば

============================
input:2d6>=9	SwordWorld
output:
SwordWorld : (2D6>=9) ＞ 8[2,6] ＞ 8 ＞ 失敗
rand:2/6,6/6
============================

は

input：（ダイスロール文字列）（対応ゲーム）
output:
（出力結果）
rand：(出目)/(振るダイスの面数),（同様に続く）

というフォーマットになります。
簡単なフォーマットですので手動で追加することも可能です。


・BCDice_forTest.rb
BCDice1.x系（本媒体の前身であるPerlバージョン）
を改造し、テストデータを生成するようにしたスクリプト
もの詳しくは事項のcreateTestData.plを参照。

・createTestData.pl
コマンドを
　perl createTestData.pl
の用に実行すると
　input.txt
(中身はtestData.txtのinput項目の羅列）
の各行を
　BCDice_forTest.rb
に渡してダイスロールを実施。
実施結果を元にテスト用のデータ
　testData_base.txt
を生成します。
この生成結果から必要なデータだけを抜粋して
　testData.txt
側に追加するのが一番手軽です。
また、
　BCDice_forTest.rb
のダイスロール結果は実際にランダムな値になりますので、
ファンブルやクリティカル等を発生させるには相応の回数を実行する必要があります。
1行は10回実行されますので、300回実行させたい場合には
　input.txt
に30行同じコマンドを記述するのが一番手っ取り早いです。


上記以外のスクリプトは内部で使うためのライブラリが大半ですので
基本的には弄らないでください。


以上です。
